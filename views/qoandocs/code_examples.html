TL;DR

A script that uses Qoan can be as simple as this:

EXAMPLE #1 use lib qw| /path/to/qoan |; use Qoan::Controller; Qoan::Controller->process_request;

A script this simple means that you must rely on Qoan's internal defaults. But what if you do not want to do that? There are several options for configuration:

Configure between instantiation and execution:

EXAMPLE #2 use lib qw| /path/to/qoan |; use Qoan::Controller; my $q = Qoan::Controller->new; # CONFIGURE QOAN HERE $q->process_request;

Some configuration options are available when Qoan is used into your script:

EXAMPLE #3 use lib qw| /path/to/qoan |; use Qoan::Controller 'main_config' => 'file.name', 'local_config' => 'file.name'; Qoan::Controller->process_request;

You can send config values as parameters to the main execution routine:

EXAMPLE #4 use lib qw| /path/to/qoan |; use Qoan::Controller; Qoan::Controller->process_request( 'config.file.name', \%config_options );

You can send config values as parameters on instantiation, too.

EXAMPLE #5 use lib qw| /path/to/qoan |; use Qoan::Controller; my $q = Qoan::Controller->new; $q->process_request( 'config.file.name', \%config_options );
